# Plan: AI Video Analysis & Caption Generation

## Goal
Add a button on the VideoEdit page that analyzes the video (frames + audio) and generates title, caption, and hashtags using Cloudflare AI.

## Architecture

### Flow
```
User clicks "AI Generate" button
  → Worker triggers GitHub Actions workflow (analyze-video.yml)
  → Workflow downloads video, extracts frames (FFmpeg), extracts audio
  → Workflow sends frames + audio to worker webhook
  → Worker: audio → Whisper → transcription
  → Worker: frames → Vision model → scene descriptions
  → Worker: web search → trending hashtags for topic
  → Worker: combines all → generates title, caption, hashtags
  → Frontend displays results in the edit form
```

### Why GitHub Actions for extraction?
- Cloudflare Workers can't run FFmpeg (CPU limit ~30s, no binary access)
- GitHub Actions has full Ubuntu with FFmpeg pre-installed
- Free tier: 2,000 minutes/month

### Models to use
1. **Whisper** (`@cf/openai/whisper`): $0.00045/min audio - transcribe speech to text
2. **Vision + Text** (`@cf/meta/llama-4-scout-17b-16e-instruct`): FREE - analyze frames + generate content

## Implementation Steps

### Step 1: Create `analyze-video.yml` workflow
**File**: `.github/workflows/analyze-video.yml`

New workflow_dispatch with inputs:
- `video_url` - URL of video to analyze
- `webhook_url` - worker callback URL
- `shortcode` - video identifier

Steps:
1. Download video via curl
2. Extract 5 key frames using FFmpeg (evenly spaced):
   ```
   ffmpeg -i video.mp4 -vf "select='not(mod(n\,floor(dur/5)))'" -vsync vfr -frames:v 5 frame_%d.jpg
   ```
3. Extract audio to WAV for Whisper:
   ```
   ffmpeg -i video.mp4 -vn -acodec pcm_s16le -ar 16000 -ac 1 audio.wav
   ```
4. Get audio duration info
5. Convert frames to base64, audio to base64
6. POST to worker webhook with: `{ shortcode, frames: [base64...], audio: base64, audio_duration }`

### Step 2: Add worker webhook endpoint
**File**: `worker/index.ts` (or new `worker/routes/ai-analysis.ts`)

New route: `POST /api/callback/analyze`

Receives:
```json
{
  "shortcode": "xxx",
  "frames": ["base64...", "base64...", "base64..."],
  "audio": "base64...",
  "audio_duration": 15
}
```

Processing:
1. **Transcribe audio** with Whisper:
   ```ts
   const audioBytes = Uint8Array.from(atob(data.audio), c => c.charCodeAt(0));
   const transcription = await env.AI.run('@cf/openai/whisper', { audio: [...audioBytes] });
   ```
2. **Analyze frames** with Vision model:
   ```ts
   const visionResult = await env.AI.run('@cf/meta/llama-4-scout-17b-16e-instruct', {
     messages: [{
       role: 'user',
       content: [
         { type: 'text', text: 'Analyze this video frame...' },
         { type: 'image', image: frameBase64 }
       ]
     }]
   });
   ```
3. **Search web** for relevant hashtags:
   - Use Cloudflare AI `web_search` tool or Worker's `fetch` to search Instagram trends
   - Query: "popular Instagram hashtags for [topic from analysis]"
   - Collect top relevant hashtags
4. **Combine results** and generate final content:
   - Input: transcription text + frame descriptions + web hashtags
   - Output: title, caption, hashtags (prioritizing trending ones from web)
5. Save to DB and return to frontend

### Step 3: Add AI analysis to video model
**File**: `worker/db/InstagramVideo.ts`

Add fields:
- `ai_analysis` - raw analysis text
- `ai_title` - generated title
- `ai_caption` - generated caption  
- `ai_hashtags` - generated hashtags

Add migration:
```sql
ALTER TABLE instagram_videos ADD COLUMN ai_analysis TEXT;
ALTER TABLE instagram_videos ADD COLUMN ai_title TEXT;
ALTER TABLE instagram_videos ADD COLUMN ai_caption TEXT;
ALTER TABLE instagram_videos ADD COLUMN ai_hashtags TEXT;
```

### Step 4: Add "AI Generate" button to VideoEdit page
**File**: `src/pages/VideoEdit.tsx`

New button: "AI تولید کپشن" (with loading spinner)
- Calls `POST /api/videos/${shortcode}/analyze`
- Shows loading state while processing
- When results arrive, fills form fields:
  - `user_caption` ← ai_caption
  - Shows ai_title and ai_hashtags in a display area

### Step 5: Add analyze endpoint to videos router
**File**: `worker/routes/videos.ts`

New route: `POST /:shortcode/analyze`

1. Get video from DB
2. Trigger `analyze-video.yml` workflow with video URL
3. Return `{ ok: true, message: 'تحلیل شروع شد' }`
4. Results arrive via webhook callback

### Step 6: Update webhook to handle analysis results
**File**: `worker/index.ts`

When analysis webhook arrives:
1. Find video by shortcode
2. Save ai_analysis, ai_title, ai_caption, ai_hashtags to DB
3. Return success
